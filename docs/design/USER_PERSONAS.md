## 2. 주요 페르소나

### Persona 1: 지민 (Ji-min) - "여행 총무"

#### 기본 정보
| 항목 | 내용 |
|------|------|
| **나이** | 28세 |
| **직업** | IT 회사 마케터 |
| **거주지** | 서울 |
| **여행 빈도** | 연 2-3회 해외여행 |

#### 프로필
> "친구들이랑 여행 가면 항상 내가 일정 짜는 역할이에요.
> 근데 카톡으로 장소 공유하면 나중에 찾기 너무 힘들어요."

#### 특성
- **디지털 활용**: 인스타그램 헤비유저, 릴스로 여행 정보 수집
- **성향**: 계획형, 꼼꼼함, 친구들 사이에서 "총무" 역할
- **기기**: iPhone 15 Pro, MacBook Air
- **SNS**: 인스타그램(매일), 유튜브(주 3-4회), 네이버 블로그(검색용)

#### Goals (목표)
1. 친구들이 보내준 장소를 한 곳에서 관리하고 싶음
2. 지도에서 위치를 보면서 효율적인 동선을 짜고 싶음
3. 친구들과 일정을 실시간으로 공유하며 의견 반영하고 싶음

#### Frustrations (불만)
1. 카카오톡으로 장소 공유하면 대화에 묻혀서 나중에 못 찾음
2. 구글맵에 저장하면 다른 친구들이 못 봄
3. 엑셀/노션으로 정리하면 지도 연동이 안 됨
4. 스크린샷으로 받은 장소는 일일이 검색해야 함

#### 기술 친숙도
```
SNS 활용:     ████████░░ 80%
새 앱 적응:   ████████░░ 80%
AI 도구:     ██████░░░░ 60%
```

#### 인용문
> "인스타에서 본 그 카페 어디더라... 스크린샷은 있는데 이름을 모르겠어"
> "얘들아 여기 어때? 하고 물어보려면 매번 링크 복사해서 보내야 해서 귀찮아"

### Persona 2: 현우 (Hyun-woo) - "효율 추구형 여행자"

#### 기본 정보
| 항목 | 내용 |
|------|------|
| **나이** | 32세 |
| **직업** | 스타트업 개발자 |
| **거주지** | 판교 |
| **여행 빈도** | 연 1-2회 (짧은 휴가) |

#### 프로필
> "휴가가 짧아서 여행 가면 최대한 효율적으로 돌아다니고 싶어요.
> AI가 알아서 최적의 일정을 짜주면 좋겠어요."

#### 특성
- **디지털 활용**: 새로운 도구/서비스 빠르게 습득
- **성향**: 효율 중시, 데이터 기반 의사결정
- **기기**: Galaxy S24 Ultra, Windows PC
- **SNS**: 유튜브(매일), X(트위터), 레딧

#### Goals (목표)
1. 여행 계획에 들이는 시간을 최소화하고 싶음
2. AI가 추천해주는 장소와 일정을 참고하고 싶음
3. 이동 동선이 최적화된 일정을 원함

#### Frustrations (불만)
1. 여행 계획 세우는 데 시간이 너무 많이 듦
2. 어디가 좋은지 일일이 검색하는 게 번거로움
3. 기존 여행 앱들은 AI 기능이 부족함

#### 기술 친숙도
```
SNS 활용:     ██████░░░░ 60%
새 앱 적응:   ██████████ 100%
AI 도구:     ████████░░ 80%
```

#### 인용문
> "ChatGPT한테 물어보면 옛날 정보 줘서 믿을 수가 없어"
> "그냥 AI가 알아서 짜주고, 내가 수정만 하면 안 되나?"

#### 기본 정보

#### 특성

- **성향**: 감성적, 비주얼 중시, FOMO(놓치기 싫음)

2. "OO 맛집" 검색해도 인스타에서 본 그 곳이 안 나옴
3. 저장한 장소가 너무 많아서 정리가 안 됨


> "이 사진 속 카페가 어딘지 알려주면 좋겠어요"

## 3. 사용자 여정 맵 (User Journey Map)

### 시나리오: 친구 4명이 함께 오사카 3박 4일 여행 계획

#### Phase 1: 인지 (Awareness)
| 단계 | 사용자 행동 | 감정 | 접점 |
|------|------------|------|------|
| 여행 결정 | 친구들과 오사카 여행 결정 | 설렘, 기대 | 카카오톡 |
| 정보 수집 시작 | 인스타/유튜브에서 오사카 여행 검색 | 흥미 | SNS |
| 문제 인식 | "이거 다 어떻게 정리하지?" | 약간의 부담 | - |

#### Phase 2: 고려 (Consideration)
| 단계 | 사용자 행동 | 감정 | 접점 |
|------|------------|------|------|
| 솔루션 탐색 | "여행 일정 앱" 검색 | 탐색 | 앱스토어, 검색 |
| Travel Planner 발견 | "스크린샷으로 장소 추출" 기능에 관심 | 호기심 | 랜딩 페이지 |
| 가입 결정 | Google 로그인으로 간편 가입 | 긍정적 | 로그인 페이지 |

#### Phase 3: 첫 사용 (First Use)
| 단계 | 사용자 행동 | 감정 | 접점 |
|------|------------|------|------|
| 프로젝트 생성 | "오사카 여행" 프로젝트 생성 | 기대 | 대시보드 |
| 스크린샷 업로드 | 인스타 스크린샷 5장 업로드 | 약간 긴장 | 입력 탭 |
| AI 분석 확인 | 장소가 자동 추출됨 확인 | 놀람, 만족 | 장소 목록 |
| 지도 확인 | 지도에서 위치 확인 | 만족 | 지도 뷰 |

#### Phase 4: 핵심 사용 (Core Usage)
| 단계 | 사용자 행동 | 감정 | 접점 |
|------|------------|------|------|
| 추가 장소 수집 | 더 많은 스크린샷/URL 추가 | 몰입 | 입력 탭 |
| AI 챗봇 활용 | "오사카 라멘 맛집 추천해줘" | 편리함 | 챗봇 |
| 일정 생성 | 3박 4일 일정 생성 | 기대 | 일정 탭 |
| 드래그앤드롭 | 장소를 Day별로 배치 | 재미 | 일정 타임라인 |
| 친구 초대 | 초대 링크로 친구 3명 초대 | 만족 | 멤버 관리 |

#### Phase 5: 공유 & 재방문 (Share & Return)
| 단계 | 사용자 행동 | 감정 | 접점 |
|------|------------|------|------|
| 일정 공유 | 완성된 일정을 공유 링크로 전송 | 뿌듯함 | 공유 페이지 |
| 친구 피드백 | 친구들이 일정 확인 및 수정 | 협업감 | 프로젝트 페이지 |
| 여행 중 참조 | 여행 중 앱으로 일정 확인 | 유용함 | 모바일 앱 |
| 재방문 | 다음 여행 계획 시 다시 사용 | 친숙함 | 대시보드 |

## 4. 핵심 시나리오

### 시나리오 1: SNS 스크린샷에서 장소 추출
```
1. 수빈이 인스타그램에서 오사카 카페 릴스를 봄
2. "여기 너무 예쁘다!" 하며 스크린샷 저장
3. Travel Planner 앱에서 스크린샷 업로드
4. AI가 "% Arabica 교토" 카페를 자동 인식
5. 지도에 위치 표시 + 상세 정보 표시
6. 일정에 바로 추가
```

### 시나리오 2: AI 챗봇으로 장소 추천 받기
```
1. 현우가 도쿄 여행 중 점심 장소 고민
2. 챗봇에 "시부야 근처 가성비 좋은 라멘집 추천해줘"
3. AI가 3곳 추천 + 이유 설명
4. 마음에 드는 곳 선택 → "일정에 추가" 버튼 클릭
5. Day 2 점심에 자동 추가됨
```

### 시나리오 3: 친구들과 함께 일정 편집
```
1. 지민이 오사카 여행 프로젝트 생성
2. 초대 링크로 친구 3명 초대
3. 각자 가고 싶은 장소 추가 (스크린샷, 검색 등)
4. 지민이 일정 초안 작성
5. 친구들이 실시간으로 확인하며 의견 제시
6. 함께 드래그앤드롭으로 순서 조정
7. 최종 일정 완성 → 공유 링크 생성
```

## 5. 디자인 임플리케이션

### 페르소나별 UI 우선순위

| 기능 | 지민 (총무) | 현우 (효율) | 수빈 (감성) |
|------|:-----------:|:-----------:|:-----------:|
| 이미지 업로드 | ★★★ | ★★☆ | ★★★ |
| AI 챗봇 | ★★☆ | ★★★ | ★★☆ |
| 지도 시각화 | ★★★ | ★★★ | ★★☆ |
| 드래그앤드롭 일정 | ★★★ | ★★☆ | ★★☆ |
| 그룹 협업 | ★★★ | ★☆☆ | ★★☆ |
| 공유 기능 | ★★★ | ★☆☆ | ★★★ |

### 핵심 디자인 원칙

1. **즉각적인 피드백**: 스크린샷 업로드 시 AI 분석 과정 시각화
2. **협업 가시성**: 누가 무엇을 추가/수정했는지 명확히 표시
3. **직관적인 조작**: 드래그앤드롭의 자연스러운 인터랙션
4. **모바일 우선 경험**: 여행 중에도 쉽게 참조할 수 있는 UI
